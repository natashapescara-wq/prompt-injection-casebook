# Contributing

Contributions are welcome if they improve clarity, safety, documentation quality, or defensive AI security understanding.

## Good Contributions

- clearer explanations
- safer examples
- better failure taxonomy
- improved evaluation rubrics
- additional mitigation notes
- links to reputable resources
- typo fixes
- beginner-friendly documentation improvements

## Avoid

- operational abuse instructions
- exploit chains against real systems
- credential theft techniques
- instructions for evading detection
- private or confidential data
- unsafe code

## Suggested Pull Request Format

1. What changed?
2. Why is it useful?
3. Does it improve safety or clarity?
4. Did you avoid including harmful operational details?
